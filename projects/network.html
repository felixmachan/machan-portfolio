Networking Home Lab Infrastructure

Overview

I built this network to avoid a pile of isolated boxes and cloud-dependent smart home apps. The WAN hits the modem, runs in bridge mode to the router, and then lands on a Lenovo ThinkCentre M710Q where services run in isolated instances. The core goal was simple: every critical function should work locally, even if the internet is down.

Problem

Home devices were scattered across separate apps and protocols. Cloud dependency, imprecise automation timing, and weak observability made the system fragile. Without a centralized, local-first backbone, it was hard to build stable, expandable automations.

Goal

The goal was a home stack where services run isolated, every device can be controlled locally, and remote access is secure. Reliability, data ownership, and a single control plane were non-negotiable.

Approach

The stack is anchored by Proxmox VE, which provides a virtualization layer so services are isolated and recoverable. The router handles traffic and internal segments, while the server hosts dedicated VMs and containers for each function.

Services and why they exist:

  • Proxmox VE - hypervisor that separates VMs and enables snapshots for quick recovery.
  • Home Assistant - the automation brain for Hue, IKEA, Sonoff, and other devices.
  • Zigbee2MQTT - stable local device communication engine for Zigbee devices without cloud dependency.
  • TrueNAS - file sharing, backups, and centralized data for projects and media.
  • Vaultwarden - secure storage for credentials and internal secrets.
  • Tailscale - secure access from outside the network over encrypted tunnels.
  • Plex - For streaming my own media to various devices.
  • RustDesk - For remote desktop access to devices within the network.
  • PiHole - Network-wide ad blocking and DNS filtering.
  • Nginx Proxy Manager - For managing reverse proxies and SSL certificates. Making local dns records to easily find my important services via a subdomain of machan.hu.

The Bambu Lab printer and Hue, IKEA, and Sonoff ecosystems are all integrated under a single control plane, keeping real-time status and automation logic in one place.

Result

A stable, maintainable, and future-proof home infrastructure where services are separated yet operate as one system, updated regularly. It delivers daily smart home reliability while supporting development and hobby projects with confidence.
